# ADDON: Аналитические модули для AsteriskTranscriptionEngine

Система аддонов — это набор аналитических инструментов поверх транскрибации звонков, реализующих gap-анализ техподдержки, ТРИЗ-анализ решений и паттерн-анализ продаж.

---

## Описание проекта

### Состав

| Модуль | Назначение | Скрипты |
|--------|-----------|---------|
| **sales_semantics/** | Анализ продаж: воронка, конверсия, приёмы | `negotiation_analyzer.py` + `sales_semantics.py` |
| **my_solutions/** | ТРИЗ-анализ решений: профиль мышления, gaps | `solutions_analyzer.py` + `solutions_semantics.py` |
| **support/** | Gap-анализ техподдержки: эмпатия, объяснение, сроки | `support_analyzer.py`, `support_semantics.py`, `quick_analyze.py`, `backend/app.py` |
| **triz/** | Bridge-анализ: LARQL vs LLM, semantic gaps | `bridge_analyzer.py` (v1–v8) |

### Принцип работы

1. **Семантический словарь** — whitelist ключевых слов/фраз, сгруппированных по концептам (например, «понимаю», «извините» → Эмпатия).
2. **Подсчёт вхождений** — контекстный поиск с учётом границ слов и кириллических окончаний.
3. **Классификация** — автоматическое определение профиля (для продаж — этап воронки, для поддержки — наличие разрывов, для решений — доминирующий ТРИЗ-концепт).
4. **Агрегация** — группировка по сотруднику, расчёт средних, топов, трендов.
5. **Генерация отчётов** — Markdown с визуализацией (ASCII-бары, таблицы, ссылки).

### Модуль продаж (`sales_semantics/`)

Анализирует транскрипты звонков по 12+ категориям: Контакт, Потребности, Возражения, Ценность, Закрытие, Риск, Координация, Спецификация, B2B-контракт.

Для каждого менеджера формируется отчёт с процентом конверсии (прокси на основе речевых маркеров), долей риск-сигналов («дорого», «подумаю», «сравню»), распределением по этапам воронки и зонами роста.

### Модуль техподдержки (`support/`)

Gap-анализ: алгоритм ищет контекстно-значимые разрывы:
- Проблема есть → эмпатии нет (разрыв)
- Проблема есть → объяснения нет (разрыв)
- Негатив есть → нормализации нет (разрыв)
- Решение есть → сроков нет (разрыв)

### Модуль ТРИЗ-решений (`my_solutions/`)

Анализирует инженерное мышление по 8 концептам: ИКР/Идеальность, Противоречие, Системность, Ресурсы, Критерии, Рефлексия. Определяет профиль мышления («ИКР-ориентированный», «Системный», «Экспресс-записи») и подсвечивает зоны роста.

## Анализ для руководства

### Дашборды

Каждый модуль генерирует иерархию отчётов:

```
addon/
├── sales_semantics/reports/
│   ├── sales_dashboard.md            ← сводка по компании
│   └── XXX_sales_report.md           ← персональный отчёт
├── support/reports/
│   ├── summary_dashboard.md          ← топ-разрывы по компании
│   └── XXX_gap_report.md             ← gap-отчёт оператора
└── my_solutions/reports/
    ├── dashboard_v2.md               ← профиль мышления команды
    └── ..._v2.md                     ← персональные отчёты
```

### Ключевые метрики

| Модуль | Метрики |
|--------|---------|
| **sales_semantics** | Конверсия (прокси), доля риск-сигналов, этап воронки, топ приёмов |
| **support** | Частота разрывов (Эмпатия, Объяснение, Нормализация), топ концептов |
| **my_solutions** | Системное покрытие (%), доминирующий профиль, плотность концептов |

### Автоинформирование

| Тип оповещения | Триггер | Канал | Модуль |
|----------------|---------|-------|--------|
| Падение конверсии | Конверсия < порога | Email / Telegram / Slack | sales_semantics |
| Всплеск риск-сигналов | Доля «Дорого/Подумаю» >20% | Email | sales_semantics |
| Системный разрыв в ТП | >50% диалогов без эмпатии | Telegram | support |
| Профиль мышления | Доминирует «Экспресс-записи» | Slack / Email | my_solutions |
| Еженедельный дашборд | Сводка по всем модулям | Email-рассылка | Все |

### Пример отчёта для руководителя продаж

```
📊 Еженедельный дашборд продаж

📈 Конверсия по компании: 12.2%
⚠️ Менеджеры с падением: XXX (8.2%), YYY (3.6%)
🔴 Риск-сигналы: 0.3% — в норме

🔥 Топ-3 менеджера недели:
  1. XXXXX — 50.0% (3 диалога)
  2. XXXXX — 50.0% (2 диалога)
  3. XXXXX — 50.0% (2 диалога)

🌱 Зоны роста:
  - XXX: Закрытие — «Как сделать следующий шаг более конкретным?»
  - YYY: Потребности — «Какой уточняющий вопрос раскроет скрытую мотивацию?»
```

### Пример отчёта для руководителя техподдержки

```
📊 Gap-анализ техподдержки

🔴 Системные разрывы:
  - Эмпатия: 34/34 операторов (100%)
  - Объяснение: 34/34 операторов (100%)

⚠️ Критические операторы:
  - XXX (85 диалогов): Эмпатия + Объяснение
  - YYY (56 диалогов): Эмпатия

✅ Рекомендуется: провести тренинг по эмпатическому слушанию,
   добавить в скрипт: «Я понимаю вашу ситуацию, давайте разберёмся»
```

### Что даёт бизнесу

- **Рост конверсии** — менеджеры получают обратную связь по конкретным речевым паттернам вместо «тайного покупателя».
- **Системное обучение ТП** — если 100% операторов не проявляют эмпатию — это пробел в скрипте, а не проблема людей. Один тренинг закрывает разрыв у всей компании.
- **Развитие мышления инженеров** — ТРИЗ-профили показывают, кто мыслит системно, а кто работает «по наитию».
- **Независимость от CRM** — аналитика работает только на аудиозаписях, без интеграции с учётными системами.
- **Экономия времени** — вместо прослушивания звонков — дашборд с метриками и рекомендациями.

### План внедрения оповещений

1. Telegram/Slack-бот — чтение Markdown-отчётов и отправка в мессенджеры
2. Email-рассылка — прикрепление дашборда через smtplib
3. Webhook — вызов внешнего API при превышении пороговых метрик

Все отчёты уже лежат в `/reports/` в структурированном Markdown, достаточно написать скрипты-обёртки.
