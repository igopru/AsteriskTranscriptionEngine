# Экспертный анализ проекта AsteriskTranscriptionEngine

## Кратко о проекте

Готовая система автоматической расшифровки телефонных звонков на базе OpenAI Whisper, интегрированная с Asterisk PBX. Включает веб-интерфейс для поиска и прослушивания записей, а также аддоны для аналитики продаж, техподдержки и ТРИЗ-коучинга.

Ключевые характеристики:
- Полный цикл: запись → транскрибация → поиск → аналитика
- Работает на CPU (GPU опционально), модель medium, INT8-квантизация
- Всё на сервере организации — данные не уходят вовне
- Веб-интерфейс с поиском по ключевым словам, маске номера и дате
- Аналитические модули для контроля качества продаж и поддержки

---

## Что это реально даёт бизнесу

**Отдел продаж.** Вместо «тайного покупателя» раз в квартал — объективная аналитика по каждому менеджеру после каждого звонка. Руководитель видит, кто из менеджеров не выявляет потребности, кто не работает с возражениями, а кто стабильно закрывает сделки. Конверсия считается не из CRM (которую можно «нарисовать»), а из речи менеджера — если он сказал «оформим», «договорились» — это маркер реального закрытия.

**Техподдержка.** Если 100% операторов не сказали «понимаю», «сожалею» или «извините» при жалобе клиента — это не 100 плохих сотрудников. Это плохой скрипт/регламент. Один тренинг, одна правка скрипта — и разрыв закрыт у всей компании за день.

**Продуктовая разработка.** Инженеры, которые мыслят системно (ИКР, противоречия, ресурсы), приносят в 3–5 раз больше ценности, чем те, кто работает «по наитию». ТРИЗ-профили показывают, кого пора развивать, а кто уже готов к сложным задачам.

## Проблема, которую решаем

| Было | Стало |
|------|-------|
| Прослушивание звонков вручную — часы работы | Поиск по тексту за секунды |
| Оценка менеджеров — субъективно («мне кажется») | Объективные метрики по каждому диалогу |
| Тренинги «для галочки» без замера результата | Gap-анализ: конкретные разрывы → конкретные навыки |
| CRM показывает «конверсию», но не показывает «почему» | Речевые маркеры показывают причину: не выявил потребность, не закрыл, клиент сомневался |
| Облачные STT — аудио уходит на сервера США/РФ | Полная локальность, 152-ФЗ, ФСБ-требования |

---

## Почему это лучше существующих решений

**vs Облачные STT (Яндекс SpeechKit, SaluteSpeech, Google/Azure STT)**
- Данные не покидают сервер — критично для банков, госорганов, медицины
- Нет поминутной оплаты — цена = железо + электричество
- Не нужен интернет
- Не нужно писать интеграцию с Asterisk — уже готова

**vs Готовые системы (Naumen, Sangoma, Voximplant)**
- Цена ниже в 5–20× при объёмах от 5 000 мин/мес
- Open source — нет vendor lock-in
- Можно дорабатывать под себя (аддоны уже это показывают)

**vs Whisper в чистом виде**
- Готовая интеграция с Asterisk
- Система очередей (watcher + worker) — не нужно писать самому
- Веб-интерфейс с поиском
- Аналитические аддоны

## Экономика

**Для клиента на 10 000 минут в месяц:**
- Облачные STT: 6 000–12 000 руб/мес
- Наше решение: сервер (аренда) ~3 000 руб/мес + электричество
- Окупаемость: 2–4 месяца

**Для интегратора/разработчика:**
- Лицензия: 50–150 тыс. руб. за установку
- Подписка на поддержку: 15–30% стоимости лицензии/год
- Аддоны — дополнительный апселл (30–70 тыс. за модуль)

---

## Технические особенности (важные для бизнеса, не для инженеров)

- **Работает на обычном сервере** — не нужен GPU за 300 тыс., хватит Xeon/Ryzen за 50–80 тыс.
- **Русский язык из коробки** — словари, токенизация, B2B-терминология РФ
- **Не требует интеграции с CRM** — ставится на чистый сервер, работает независимо
- **Справляется с NFS** — Asterisk может писать на сетевую папку, это не проблема
- **Восстанавливается после сбоев** — журнал обработанных файлов, не транскрибирует заново
- **Масштабируется горизонтально** — не хватило одного worker → запустили второй

## Риски и ограничения

- **Точность модели** — medium-модель ошибается на специфичной лексике (иностранные слова, фамилии). Для юридически значимой транскрибации требуется large-v3 + GPU.
- **Отсутствие диаризации** — пока не разделяет голоса (кто сказал «здравствуйте» — менеджер или клиент). WhisperX эту задачу решает.
- **Ручная настройка словарей** — whitelist поддерживается вручную, требуется актуализация при изменении скриптов продаж.
- **Конкуренция** — рынок STT быстро дешевеет. Через 1–2 года облачные сервисы могут стать дешевле собственного сервера. Защита: упор на локализацию и конфиденциальность.

---

## Стратегический вывод

Проект занимает нишу, которую облачные вендоры не могут закрыть по определению: **полностью локальная, конфиденциальная, не требующая интеграции система анализа звонков для среднего бизнеса в РФ**. Дополнительные аддоны (аналитика продаж, gap-анализ ТП, ТРИЗ-коучинг) создают дифференциацию от «голой» транскрибации и превращают продукт в комплексное решение для управления качеством коммуникаций.

Основной вектор развития: добавление диаризации (кто говорит), увеличение точности (large-v3), расширение библиотеки словарей под разные отрасли (медицина, логистика, ритейл), автоматизация оповещений.
