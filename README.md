# AsteriskTranscriptionEngine

Система автоматической расшифровки телефонных разговоров (speech-to-text) для Asterisk PBX на базе OpenAI Whisper (faster-whisper) с веб-интерфейсом для поиска и прослушивания записей.

## Техническое описание

Гибридная система реального времени + пакетной обработки, работающая по схеме **producer-consumer**:

- **watcher.py** — демон, поллингующий NFS-директорию `/monitor/YYYY/MM/DD/`, куда Asterisk складывает WAV-файлы звонков. Классифицирует файлы по приоритету (на основе regex-маски номера) и помещает в две файловые очереди: `priority_queue.txt` и `backfill_queue.txt`. Конкурентный доступ защищён `filelock`.
- **worker.py** — демон, загружающий модель `faster-whisper medium` в RAM, забирающий файлы из очередей (приоритетная — первой), нормализующий аудио через FFmpeg (loudnorm EBU R128, 16 кГц, моно, PCM S16LE) и транскрибирующий в UTF-8 текст. Результат сохраняется как `.wav.txt` рядом с оригиналом.
- **wav2txt.py** — пакетный обработчик для транскрибации исторических записей за произвольный период, запускаемый через веб-интерфейс.
- **Веб-приложение (Flask)** — поиск записей по дате, маске номера, ключевым словам (с wildcard-символами), прослушивание записей через expiring token (itsdangerous), запуск/мониторинг пакетной обработки.

### Стек технологий

| Компонент | Технология |
|-----------|-----------|
| Модель STT | faster-whisper (CTranslate2, medium, Russian) |
| Аудиообработка | FFmpeg (loudnorm, 16 кГц, моно) |
| Веб-фреймворк | Flask 2.3 + Gunicorn |
| Фронтенд | Bootstrap 4, jQuery, HTML5 Audio |
| Безопасность | Session-based auth, expiring signed URLs |
| Очереди | Файловые (.txt) + filelock |
| Запуск | systemd (whisper-watcher, whisper-worker) |

### Продукт снабжен подробным инженерным Runbook для адаптации к архитектуре Tiered Storage (уровневого хранения) заказчика на базе любого сетевого оборудования (NFS/SSHFS/NAS).

### Из чего состоит

- `/watcher.py`, `/worker.py`, `/rt_config.cfg` — компоненты реального времени
- `web/app.py` — Flask-приложение (поиск, конфиг, пакетная обработка)
- `web/templates/` — шаблоны Bootstrap (login, index, config, 404, 500)
- `AsteriskTranscriptionEngine/whisper3/wav2txt.py` — пакетный транскрибатор

## Принцип новизны

1. **Приоритезация очереди на основе номера телефона** — файлы звонков с определённых масок номеров обрабатываются вне очереди. Позволяет гарантировать минимальную задержку транскрибации для критичных абонентов без дублирования инфраструктуры.
2. **Файловые очереди с блокировками** — отказ от Redis/RabbitMQ в пользу plain-text файлов + filelock. Система не требует внешних сервисов, восстанавливается после перезагрузки (журнал `processed_files.txt`), прозрачна для оператора.
3. **Polling вместо inotify** — осознанный отказ от файловых уведомлений (inotify не работает через NFS). Поллинг каждые 5 секунд с проверкой mtime > 3 сек (чтобы не хватать недописанный файл) — надёжное решение для NFS-окружений.
4. **Единая кодовая база для real-time и batch** — `worker.py` и `wav2txt.py` используют идентичный пайплайн нормализации + транскрибации, что исключает расхождение результатов.
5. **Защита интеллектуальной собственности** — документация предусматривает компиляцию исходников (Nuitka/PyArmor) или поставку в Docker-образе без доступа к `.py` файлам, что позволяет коммерческое распространение без раскрытия кода.

## Зачем это нужно

- **Контроль качества обслуживания** — автоматический анализ звонков без участия человека, поиск по ключевым словам (жалобы, маркеры скриптов продаж).
- **Исполнение требований регуляторов** — в ряде отраслей (банки, страхование, телемедицина) требуется архивировать и индексировать телефонные переговоры.
- **Интеграция с CRM/BI** — текстовая расшифровка звонков пригодна для автоматической классификации, извлечения сущностей (суммы, даты, имена), построения дашбордов.
- **Экономия времени** — поиск по звонкам становится быстрым как Ctrl+F, не нужно прослушивать часы записей.
- **Локальная установка без облака** — все данные остаются на сервере организации, нет утечки аудиозаписей вовне.

## Коммерческий потенциал

### Модель монетизации

1. **Продажа лицензии на ПО** — единоразовая плата за установку под ключ: серверная часть + веб-интерфейс + настройка интеграции с Asterisk заказчика. Целевая аудитория: малый и средний бизнес (от 20 до 500 сотрудников), отделы продаж, контакт-центры, тендерные организации.
2. **Ежегодная подписка на обновления и поддержку** — новые версии модели Whisper (large-v3, future), доработки под конкретные маски номеров, обновления безопасности, техподдержка по e-mail/Telegram.
3. **White-label / Private-label** — ребрендинг под корпоративный стиль заказчика (логотип, название, цветовая схема). Отдельный прайс — от N копий.
4. **SaaS-модель (аренда)** — размещение на сервере интегратора, оплата помесячно за количество протранскрибированных минут. Для клиентов, не желающих администрировать свой сервер.
5. **Доработки за отдельную плату** — интеграция с SIP-транками напрямую (без файлов), диаризация (WhisperX), дашборды в Grafana, интеграция с CRM (amoCRM, Bitrix24), телеграм-бот оповещений.
6. **Аппаратный модуль** — поставка в виде готового сервера 1U (Intel NUC / Mini PC) с предустановленной системой, настройкой и гарантией. Цена = железо + софт + настройка.

### Конкурентные преимущества для коммерциализации

- **Цена в 5–20× ниже облачных сервисов** при объёмах от 5 000 мин/мес (окупается за 2–3 месяца).
- **Данные не покидают сервер клиента** — критически важно для банков, госорганов, медицинских учреждений.
- **Единый интерфейс** для RT-транскрибации, исторической обработки и поиска — конкуренты (облачные STT) дают только API, UI нужно писать отдельно.
- **Полный opensource-стек** — не требуется закупать лицензии Microsoft/Oracle, нет vendor lock-in.

### Оценка рынка (РФ, 2026)

- Рынок речевых технологий в РФ — ~15–20 млрд руб/год, сегмент телефонии — ~3–5 млрд.
- Основные драйверы: 152-ФЗ (локализация данных), 59-ФЗ (запись обращений), тренд на импортозамещение и уход западных вендоров.
- Ценовой ориентир: лицензия — 50–150 тыс. руб. (зависит от числа номеров), подписка — 15–30% от стоимости лицензии/год.

### Бесплатные / Open Source

| Продукт | Особенности |
|---------|-------------|
| **OpenAI Whisper (original)** | Базовая модель OpenAI, требует GPU+большой RAM, медленнее faster-whisper, точность выше на русском. Есть `large-v3` с наилучшим качеством. |
| **faster-whisper** (используется здесь) | **CTranslate2-оптимизация Whisper — в 4× быстрее, в 2× меньше памяти.** Наш стек. Поддерживает INT8, CPU/GPU, device auto. |
| **WhisperX** | Faster-whisper + forced alignment + VAD + diarization. Добавляет определение говорящих (AI-диаризация), что актуально для звонков двух абонентов. |
| **Vosk** | Apache 2.0, работает на CPU, маленький размер модели (~50 МБ), но точность на русском языке заметно ниже Whisper. Хорош для простых команд, не для разговорной речи. |
| **Coqui STT** | Open-source STT, есть русская модель, но проект заморожен, точность ниже Whisper. |
| **wav2vec 2.0 (Meta/HuggingFace)** | Модели от Facebook, fine-tune под свой язык. Требует глубоких знаний ML, не out-of-the-box. |
